Writing Services

Learn how to implement business logic in the Service layer

What is a Service?

A Service is a class that contains your application's business logic. Services sit between Controllers and Repositories, orchestrating complex operations, enforcing business rules, and managing database transactions.

Three-Layer Architecture

Controller Layer
  • Handles HTTP requests
  • Validates input format
  • Calls Service methods
  • Returns formatted responses
Service Layer (YOU ARE HERE)
  • Contains business logic and rules
  • Validates business constraints
  • Orchestrates multiple repository calls
  • Manages database transactions
  • Throws business exceptions
Repository Layer
  • Executes database queries
  • Maps data to arrays
  • No business logic

Why Services Exist

Example: Creating an Order

Without Services (Bad): Controller would need to:

  • Validate user ID exists
  • Check each product exists and has enough stock
  • Calculate subtotals for each item
  • Calculate total order amount
  • Start a database transaction
  • Insert order record
  • Insert order items
  • Reduce stock for each product
  • Log the transaction
  • Commit or rollback on error

With Services (Good): Controller just calls:

$order = $this->orderService->createOrder($orderData, $items);

All the complex logic is encapsulated in the service!

Service Layer Responsibilities

✓ Services SHOULD

Implement business rules and validation
Orchestrate multiple repository calls
Handle database transactions
Calculate derived values (totals, discounts, etc.)
Validate business constraints (stock availability, order status, etc.)
Throw meaningful exceptions (ValidationException, NotFoundException)
Return structured data (arrays/objects)

✗ Services SHOULD NOT

Parse HTTP requests or format responses
Write SQL queries directly
Return HTTP response objects
Access $_GET, $_POST, or $_SERVER directly
Handle routing or middleware logic

Creating Your First Service

File Location and Naming

  • Location: src/api/services/
  • Naming: *Service.php (e.g., ProductService.php)
  • Class Name: Must match filename (e.g., class ProductService)

Basic Service Template

<?php
// src/api/services/ProductService.php

namespace App\Services;

use App\Core\Database;
use App\Core\Exceptions\ValidationException;
use App\Core\Exceptions\NotFoundException;
use App\Repositories\ProductRepository;

class ProductService
{
    private Database $db;
    private ProductRepository $productRepo;
    
    /**
     * Constructor - Accept Database instance
     */
    public function __construct(Database $db)
    {
        $this->db = $db;
        
        // Initialize repositories
        $this->productRepo = new ProductRepository($db);
    }
    
    // Your business logic methods go here
    
    public function getProducts(array $filters): array
    {
        // Business logic for fetching products
    }
    
    public function getProduct(int $productId): array
    {
        // Business logic for single product
    }
    
    public function createProduct(array $data): array
    {
        // Business logic for creating product
    }
}

Constructor Pattern

Services always accept a Database instance in the constructor. This allows:

  • Passing the database connection from the controller
  • Initializing repositories with the same database instance
  • Using transactions across multiple repositories

Using Repositories in Services

Services use repositories to access data. Never write SQL queries directly in services.

Repository Initialization

class OrderService
{
    private Database $db;
    private ProductRepository $productRepo;
    private OrderRepository $orderRepo;
    
    public function __construct(Database $db)
    {
        $this->db = $db;
        
        // Initialize all repositories you need
        $this->productRepo = new ProductRepository($db);
        $this->orderRepo = new OrderRepository($db);
    }
}

Calling Repository Methods

public function getProduct(int $productId): array
{
    // Call repository method
    $product = $this->productRepo->findById($productId);
    
    // Check if found
    if (!$product) {
        throw new NotFoundException("Product ID {$productId} not found");
    }
    
    // Return the data
    return $product;
}

Transaction Handling

Transactions ensure that multiple database operations either all succeed or all fail. This prevents data inconsistency.

When to Use Transactions

Use transactions when you need to perform multiple related database operations:

  • Creating an order with multiple items
  • Transferring money between accounts
  • Updating inventory and creating audit logs
  • Any operation where partial completion would corrupt data

Transaction Syntax

// Basic transaction pattern
return $this->db->transaction(function($db) use ($data) {
    // Step 1: Database operation
    $orderId = $this->orderRepo->create($data);
    
    // Step 2: Another database operation
    $this->orderRepo->addOrderItem($itemData);
    
    // Step 3: Yet another operation
    $this->productRepo->reduceStock($productId, $quantity);
    
    // If we reach here, transaction commits automatically
    return $orderId;
});
How Transactions Work
1. BEGIN TRANSACTION - Start tracking changes
2. Execute Operations - Perform database queries
3a. COMMIT - If all succeed, save changes permanently
3b. ROLLBACK - If any fails, undo all changes

Complete Service Examples

ProductService - Full Implementation

<?php

namespace App\Services;

use App\Core\Database;
use App\Core\Exceptions\ValidationException;
use App\Core\Exceptions\NotFoundException;
use App\Repositories\ProductRepository;

/**
 * ProductService - Business logic for product management
 */
class ProductService
{
    private Database $db;
    private ProductRepository $productRepo;
    
    public function __construct(Database $db)
    {
        $this->db = $db;
        $this->productRepo = new ProductRepository($db);
    }
    
    /**
     * Get all products with filters
     * 
     * @param array $filters Filter criteria
     * @return array Products list
     */
    public function getProducts(array $filters): array
    {
        $category = $filters['category'] ?? null;
        $limit = $filters['limit'] ?? 10;
        
        return $this->productRepo->getAll($category, $limit);
    }
    
    /**
     * Get product by ID
     * 
     * @param int $productId Product ID
     * @return array Product data
     * @throws NotFoundException If product not found
     */
    public function getProduct(int $productId): array
    {
        $product = $this->productRepo->findById($productId);
        
        if (!$product) {
            throw new NotFoundException("Product ID {$productId} not found");
        }
        
        return $product;
    }
    
    /**
     * Create new product
     * 
     * @param array $data Product data
     * @return array Created product
     * @throws ValidationException On validation error
     */
    public function createProduct(array $data): array
    {
        // Validate business rules
        $this->validateProductData($data);
        
        // Create product
        $productId = $this->productRepo->create($data);
        
        // Return created product
        return $this->productRepo->findById($productId);
    }
    
    /**
     * Update product
     * 
     * @param int $productId Product ID
     * @param array $data Data to update
     * @return array Updated product
     * @throws NotFoundException If product not found
     * @throws ValidationException On validation error
     */
    public function updateProduct(int $productId, array $data): array
    {
        // Check product exists
        $product = $this->productRepo->findById($productId);
        if (!$product) {
            throw new NotFoundException("Product ID {$productId} not found");
        }
        
        // Update
        $updated = $this->productRepo->update($productId, $data);
        
        if (!$updated) {
            throw new ValidationException("Failed to update product");
        }
        
        // Return updated product
        return $this->productRepo->findById($productId);
    }
    
    /**
     * Validate product data (Business Rules)
     * 
     * @param array $data Product data
     * @throws ValidationException On validation failure
     */
    private function validateProductData(array $data): void
    {
        if (empty($data['name'])) {
            throw new ValidationException("Product name is required");
        }
        
        if (empty($data['sku'])) {
            throw new ValidationException("Product SKU is required");
        }
        
        if (!isset($data['price']) || $data['price'] <= 0) {
            throw new ValidationException("Valid product price is required");
        }
    }
}

OrderService - Full Implementation with Transactions

<?php

namespace App\Services;

use App\Core\Database;
use App\Core\Exceptions\ValidationException;
use App\Core\Exceptions\NotFoundException;
use App\Core\Exceptions\DatabaseException;
use App\Repositories\OrderRepository;
use App\Repositories\ProductRepository;

/**
 * OrderService - Business logic for order management
 * Handles order creation with transactions to ensure data consistency
 */
class OrderService
{
    private Database $db;
    private ProductRepository $productRepo;
    private OrderRepository $orderRepo;
    
    public function __construct(Database $db)
    {
        $this->db = $db;
        $this->productRepo = new ProductRepository($db);
        $this->orderRepo = new OrderRepository($db);
    }
    
    /**
     * Create a new order with items
     * Uses transaction to ensure atomicity
     * 
     * @param array $orderData Order data
     * @param array $items Order items
     * @return array Created order with details
     * @throws ValidationException On validation or database errors
     */
    public function createOrder(array $orderData, array $items): array
    {
        // Validate input
        $this->validateOrderData($orderData, $items);
        
        try {
            // Start transaction
            return $this->db->transaction(function($db) use ($orderData, $items) {
                
                // Step 1: Create the order
                $orderId = $this->orderRepo->create([
                    'userid' => $orderData['userid'],
                    'totalamount' => $orderData['totalamount'],
                    'paymentmethod' => $orderData['paymentmethod'],
                    'status' => 'pending',
                    'notes' => $orderData['notes'] ?? null
                ]);
                
                // Step 2: Process each order item
                $orderItems = [];
                foreach ($items as $item) {
                    // Verify product exists and has enough stock
                    $product = $this->productRepo->findById($item['productid']);
                    
                    if (!$product) {
                        throw new NotFoundException("Product ID {$item['productid']} not found");
                    }
                    
                    if ($product['stock'] < $item['quantity']) {
                        throw new ValidationException(
                            "Insufficient stock for product '{$product['name']}'. " .
                            "Available: {$product['stock']}, Requested: {$item['quantity']}"
                        );
                    }
                    
                    // Calculate subtotal
                    $subtotal = $product['price'] * $item['quantity'];
                    
                    // Add order item
                    $itemId = $this->orderRepo->addOrderItem([
                        'orderid' => $orderId,
                        'productid' => $item['productid'],
                        'quantity' => $item['quantity'],
                        'price' => $product['price'],
                        'subtotal' => $subtotal
                    ]);
                    
                    // Reduce product stock
                    $this->productRepo->reduceStock($item['productid'], $item['quantity']);
                    
                    $orderItems[] = [
                        'id' => $itemId,
                        'productid' => $item['productid'],
                        'productname' => $product['name'],
                        'quantity' => $item['quantity'],
                        'price' => $product['price'],
                        'subtotal' => $subtotal
                    ];
                }
                
                // Step 3: Log the transaction
                $this->orderRepo->createTransactionLog([
                    'orderid' => $orderId,
                    'action' => 'order_created',
                    'amount' => $orderData['totalamount'],
                    'paymentmethod' => $orderData['paymentmethod'],
                    'status' => 'success',
                    'notes' => 'Order created successfully'
                ]);
                
                // Return complete order data
                return [
                    'orderid' => $orderId,
                    'userid' => $orderData['userid'],
                    'totalamount' => $orderData['totalamount'],
                    'paymentmethod' => $orderData['paymentmethod'],
                    'status' => 'pending',
                    'items' => $orderItems,
                    'createdat' => date('Y-m-d H:i:s')
                ];
            });
            
        } catch (DatabaseException $e) {
            throw new ValidationException("Database error while creating order: " . $e->getMessage());
        } catch (\Exception $e) {
            throw new ValidationException("Failed to create order: " . $e->getMessage());
        }
    }
    
    /**
     * Cancel an order and restore stock
     * 
     * @param int $orderId Order ID
     * @param string $reason Cancellation reason
     * @return bool Success status
     * @throws ValidationException On error
     */
    public function cancelOrder(int $orderId, string $reason): bool
    {
        try {
            return $this->db->transaction(function($db) use ($orderId, $reason) {
                
                // Get order
                $order = $this->orderRepo->findById($orderId);
                
                if (!$order) {
                    throw new NotFoundException("Order ID {$orderId} not found");
                }
                
                if ($order['status'] === 'cancelled') {
                    throw new ValidationException("Order is already cancelled");
                }
                
                if ($order['status'] === 'completed') {
                    throw new ValidationException("Cannot cancel completed order");
                }
                
                // Get order items
                $items = $this->orderRepo->getOrderItems($orderId);
                
                // Restore stock for each item
                foreach ($items as $item) {
                    $this->productRepo->increaseStock($item['productid'], $item['quantity']);
                }
                
                // Update order status
                $this->orderRepo->update($orderId, [
                    'status' => 'cancelled',
                    'cancellationreason' => $reason,
                    'cancelledat' => date('Y-m-d H:i:s')
                ]);
                
                // Log cancellation
                $this->orderRepo->createTransactionLog([
                    'orderid' => $orderId,
                    'action' => 'order_cancelled',
                    'amount' => $order['totalamount'],
                    'paymentmethod' => $order['paymentmethod'],
                    'status' => 'refunded',
                    'notes' => "Cancellation reason: {$reason}"
                ]);
                
                return true;
            });
            
        } catch (\Exception $e) {
            throw new ValidationException("Failed to cancel order: " . $e->getMessage());
        }
    }
    
    /**
     * Get order details with items
     * 
     * @param int $orderId Order ID
     * @return array Order details
     * @throws NotFoundException If order not found
     */
    public function getOrderDetails(int $orderId): array
    {
        $order = $this->orderRepo->findById($orderId);
        
        if (!$order) {
            throw new NotFoundException("Order ID {$orderId} not found");
        }
        
        $items = $this->orderRepo->getOrderItems($orderId);
        $transactions = $this->orderRepo->getTransactionLog($orderId);
        
        return [
            'order' => $order,
            'items' => $items,
            'transactions' => $transactions
        ];
    }
    
    /**
     * Validate order data (Business Rules)
     * 
     * @param array $orderData Order data
     * @param array $items Order items
     * @throws ValidationException On validation failure
     */
    private function validateOrderData(array $orderData, array $items): void
    {
        // Validate required fields
        if (empty($orderData['userid'])) {
            throw new ValidationException("User ID is required");
        }
        
        if (empty($orderData['paymentmethod'])) {
            throw new ValidationException("Payment method is required");
        }
        
        if (!in_array($orderData['paymentmethod'], ['card', 'cash', 'upi'])) {
            throw new ValidationException("Invalid payment method");
        }
        
        if (empty($items) || !is_array($items)) {
            throw new ValidationException("Order must have at least one item");
        }
        
        // Validate items
        foreach ($items as $item) {
            if (empty($item['productid'])) {
                throw new ValidationException("Product ID is required for all items");
            }
            
            if (empty($item['quantity']) || $item['quantity'] < 1) {
                throw new ValidationException("Valid quantity is required for all items");
            }
        }
    }
}

Exception Handling in Services

Services should throw meaningful exceptions that controllers can catch and handle appropriately.

Exception Types

Exception When to Throw Example
NotFoundException Resource doesn't exist Product ID not found
ValidationException Business rule violation Insufficient stock, invalid status
DatabaseException Database operation failed Query error, connection failure

Exception Examples

// NotFoundException - Resource not found
$product = $this->productRepo->findById($productId);
if (!$product) {
    throw new NotFoundException("Product ID {$productId} not found");
}

// ValidationException - Business rule violation
if ($product['stock'] < $quantity) {
    throw new ValidationException(
        "Insufficient stock. Available: {$product['stock']}, Requested: {$quantity}"
    );
}

// ValidationException with errors array
if (!$this->isValidEmail($email)) {
    throw new ValidationException("Validation failed", [
        'email' => 'Invalid email format'
    ]);
}

// DatabaseException - Database failure
if (!$this->productRepo->reduceStock($productId, $quantity)) {
    throw new DatabaseException(
        "Failed to reduce stock for product ID {$productId}"
    );
}

Business Logic Validation

Services validate business rules, not just input format. This is different from controller validation.

Controller Validation vs Service Validation

Controller Validation (Input Format):

// Is the data present and in the right format?
if (empty($data['productid'])) {
    $this->sendError('Product ID is required');
}

if (!is_numeric($data['quantity'])) {
    $this->sendError('Quantity must be a number');
}

Service Validation (Business Rules):

// Does the product exist?
$product = $this->productRepo->findById($productId);
if (!$product) {
    throw new NotFoundException("Product not found");
}

// Is there enough stock?
if ($product['stock'] < $quantity) {
    throw new ValidationException("Insufficient stock");
}

// Is the product active?
if ($product['status'] !== 'active') {
    throw new ValidationException("Product is not available for purchase");
}

Service Best Practices

✓ DO

Use transactions for multi-step operations
Validate business rules, not just input format
Throw specific exceptions (NotFoundException, ValidationException)
Keep services focused on one domain (OrderService, ProductService)
Return structured data (arrays/objects)
Initialize repositories in the constructor
Document complex business logic with comments
Use private methods for internal validation

✗ DON'T

Write SQL queries in services (use repositories)
Access HTTP request data directly ($_POST, $_GET)
Return HTTP response objects
Skip transaction handling for multi-step operations
Catch exceptions and hide them (let them propagate)
Mix multiple domain concerns in one service
Perform simple CRUD without validation (that's what repositories are for)

Common Service Patterns

Pattern 1: Create with Validation

public function createProduct(array $data): array
{
    // 1. Validate business rules
    $this->validateProductData($data);
    
    // 2. Create via repository
    $productId = $this->productRepo->create($data);
    
    // 3. Return created resource
    return $this->productRepo->findById($productId);
}

Pattern 2: Update with Existence Check

public function updateProduct(int $productId, array $data): array
{
    // 1. Check existence
    $product = $this->productRepo->findById($productId);
    if (!$product) {
        throw new NotFoundException("Product not found");
    }
    
    // 2. Validate changes
    $this->validateProductData($data);
    
    // 3. Update
    $this->productRepo->update($productId, $data);
    
    // 4. Return updated resource
    return $this->productRepo->findById($productId);
}

Pattern 3: Multi-Repository Transaction

public function transferStock(int $fromProductId, int $toProductId, int $quantity): bool
{
    return $this->db->transaction(function($db) use ($fromProductId, $toProductId, $quantity) {
        
        // Check source product has enough stock
        $fromProduct = $this->productRepo->findById($fromProductId);
        if ($fromProduct['stock'] < $quantity) {
            throw new ValidationException("Insufficient stock in source product");
        }
        
        // Check destination product exists
        $toProduct = $this->productRepo->findById($toProductId);
        if (!$toProduct) {
            throw new NotFoundException("Destination product not found");
        }
        
        // Perform transfer
        $this->productRepo->reduceStock($fromProductId, $quantity);
        $this->productRepo->increaseStock($toProductId, $quantity);
        
        return true;
    });
}

Pattern 4: Complex Business Logic

public function applyDiscount(int $orderId, string $couponCode): array
{
    // 1. Get order
    $order = $this->orderRepo->findById($orderId);
    if (!$order) {
        throw new NotFoundException("Order not found");
    }
    
    // 2. Validate order status
    if ($order['status'] !== 'pending') {
        throw new ValidationException("Discount can only be applied to pending orders");
    }
    
    // 3. Validate coupon
    $coupon = $this->couponRepo->findByCode($couponCode);
    if (!$coupon) {
        throw new ValidationException("Invalid coupon code");
    }
    
    if (!$this->isCouponValid($coupon)) {
        throw new ValidationException("Coupon has expired");
    }
    
    // 4. Calculate discount
    $discount = $this->calculateDiscount($order['totalamount'], $coupon);
    
    // 5. Update order
    return $this->db->transaction(function($db) use ($orderId, $discount, $couponCode) {
        $this->orderRepo->update($orderId, [
            'discount' => $discount,
            'couponcode' => $couponCode,
            'totalamount' => $order['totalamount'] - $discount
        ]);
        
        $this->couponRepo->markAsUsed($couponCode);
        
        return $this->orderRepo->findById($orderId);
    });
}

Transaction Best Practices

Important Transaction Rules

  • Use transactions for multi-step operations - If step 2 fails, step 1 should be undone
  • Keep transactions short - Long transactions can cause database locks
  • Don't call external APIs inside transactions - External calls can fail and rollback everything
  • Let exceptions propagate - If an exception occurs, the transaction automatically rolls back
  • Return data from transactions - The return value of the callback is returned to the caller

Transaction Anti-Patterns (Avoid These)

// ❌ BAD: Catching exceptions inside transaction
return $this->db->transaction(function($db) {
    try {
        $this->orderRepo->create($data);
        $this->productRepo->reduceStock($id, $qty);
    } catch (\Exception $e) {
        // Don't catch here! Let it propagate so transaction rolls back
        error_log($e->getMessage());
    }
});

// ✓ GOOD: Let exceptions propagate
return $this->db->transaction(function($db) {
    $this->orderRepo->create($data);
    $this->productRepo->reduceStock($id, $qty);
    // If exception occurs, transaction automatically rolls back
});

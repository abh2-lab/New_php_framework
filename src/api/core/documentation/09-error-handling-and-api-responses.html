Error Handling & API Responses

This framework provides a consistent, structured approach to error handling and API responses.

All controllers inherit response methods from BaseController, and all exceptions are caught by the global ExceptionHandler which formats them as JSON.

Response Philosophy

The framework follows these principles:

  • Consistent Structure: All API responses follow the same JSON format
  • Proper HTTP Status Codes: Use correct status codes for different scenarios
  • Exception-Based Errors: Throw exceptions, don't return error arrays
  • Global Exception Handler: Catch all uncaught exceptions and format them
  • Debug vs Production: Show details in development, hide in production

BaseController Response Methods

All controllers extend BaseController, which provides these methods:

1. sendSuccess() - Success Response

200 OK 
protected function sendSuccess(
    string $message, 
    $data = null, 
    array $extra = []
): void

// Usage in controller:
$this->sendSuccess('Order created successfully', [
    'order_id' => 123,
    'total' => 299.99
]);
Response (200): 
{
  "status": "success",
  "message": "Order created successfully",
  "data": {
    "order_id": 123,
    "total": 299.99
  }
}

2. sendError() - Generic Error Response

400-500 
protected function sendError(
    string $message, 
    int $statusCode = 400, 
    $data = null
): void

// Usage:
$this->sendError('Product not found', 404);
Response (404): 
{
  "status": "error",
  "message": "Product not found"
}

3. sendValidationError() - Validation Errors

422 Unprocessable Entity 
protected function sendValidationError(
    string $message, 
    array $errors
): void

// Usage:
$this->sendValidationError('Validation failed', [
    'email' => 'Email is required',
    'password' => 'Password must be at least 8 characters'
]);
Response (422): 
{
  "status": "error",
  "message": "Validation failed",
  "data": {
    "validation_errors": {
      "email": "Email is required",
      "password": "Password must be at least 8 characters"
    }
  }
}

4. sendServerError() - Internal Server Error

500 Internal Server Error 
protected function sendServerError(string $message): void

// Usage:
$this->sendServerError('Database connection failed');
Response (500): 
{
  "status": "error",
  "message": "Database connection failed"
}

Framework Exception Classes

BaseException (Abstract)

All framework exceptions extend BaseException:

// File: src/api/core/exceptions/BaseException.php

abstract class BaseException extends \Exception {
    protected int $httpStatusCode = 500;
    protected string $errorType = 'error';
    protected array $data = [];

    public function getHttpStatusCode(): int {
        return $this->httpStatusCode;
    }

    public function toArray(): array {
        $response = [
            'status' => 'error',
            'error_type' => $this->errorType,
            'message' => $this->getMessage()
        ];

        if (!empty($this->data)) {
            $response['data'] = $this->data;
        }

        return $response;
    }
}

NotFoundException

404 Not Found 
// File: src/api/core/exceptions/NotFoundException.php

class NotFoundException extends BaseException {
    protected int $httpStatusCode = 404;
    protected string $errorType = 'not_found';
}

// Usage:
throw new NotFoundException('Product with ID 999 not found');
Response (404): 
{
  "status": "error",
  "error_type": "not_found",
  "message": "Product with ID 999 not found"
}

ValidationException

422 Unprocessable Entity 
// File: src/api/core/exceptions/ValidationException.php

class ValidationException extends BaseException {
    protected int $httpStatusCode = 422;
    protected string $errorType = 'validation_error';

    public function __construct(string $message, array $errors = []) {
        parent::__construct($message);
        $this->data = ['validation_errors' => $errors];
    }
}

// Usage:
throw new ValidationException('Invalid input data', [
    'email' => 'Email format is invalid',
    'age' => 'Age must be a number'
]);
Response (422): 
{
  "status": "error",
  "error_type": "validation_error",
  "message": "Invalid input data",
  "data": {
    "validation_errors": {
      "email": "Email format is invalid",
      "age": "Age must be a number"
    }
  }
}

DatabaseException

500 Internal Server Error 
// File: src/api/core/exceptions/DatabaseException.php

class DatabaseException extends BaseException {
    protected int $httpStatusCode = 500;
    protected string $errorType = 'database_error';

    public function __construct(string $message, ...) {
        // In production, hide detailed DB errors
        if (!\$_ENV['DEBUG_MODE']) {
            $message = 'Database error occurred';
        }
        parent::__construct($message);
    }
}

// Usage: (Usually thrown by Database class automatically)
throw new DatabaseException('Query failed: Table does not exist');

Global ExceptionHandler

The ExceptionHandler is registered in index.php and catches all uncaught exceptions:

// File: src/api/index.php

// Register global exception handler
ExceptionHandler::register();

How It Works

  1. Catches any uncaught exception
  2. Checks if it's a framework exception (BaseException)
  3. Formats it as JSON with proper status code
  4. Logs error if LOG_ERRORS=true
  5. Shows debug info if SHOW_ERRORS=true

Debug Mode vs Production

Debug Mode (SHOW_ERRORS=true)

{
  "status": "error",
  "error_type": "database_error",
  "message": "Query failed: Unknown column 'foo' in 'field list'",
  "debug_info": {
    "exception_class": "App\Core\Exceptions\DatabaseException",
    "file": "/src/api/core/Database.php",
    "line": 45,
    "stack_trace": [...]
  }
}

Production Mode (SHOW_ERRORS=false)

{
  "status": "error",
  "error_type": "database_error",
  "message": "Database error occurred"
}

Real Framework Examples

Example 1: ProductController - Product Not Found

// File: src/api/controllers/ProductController.php

public function details() {
    try {
        $productId = $_GET['product_id'] ?? null;

        if (!$productId) {
            $this->sendError('Product ID is required', 400);
            return;
        }

        $product = $this->productRepo->findById((int) $productId);

        if (!$product) {
            // Throw NotFoundException - caught by ExceptionHandler
            throw new NotFoundException("Product with ID {$productId} not found");
        }

        $this->sendSuccess('Product found', $product);

    } catch (\Exception $e) {
        $this->sendServerError($e->getMessage());
    }
}

Example 2: OrderController - Validation Errors

// File: src/api/controllers/OrderController.php

public function create() {
    try {
        $data = $this->getRequestData();

        // Validate using helper method
        $errors = $this->validateRequest([
            'user_id' => 'required|integer',
            'payment_method' => 'required|in:card,cash,upi',
            'items' => 'required|array'
        ]);

        if (!empty($errors)) {
            // Throw ValidationException
            throw new ValidationException('Validation failed', $errors);
        }

        // Process order via service
        $orderId = $this->orderService->createOrder($data);

        $this->sendSuccess('Order created successfully', [
            'order_id' => $orderId
        ]);

    } catch (ValidationException $e) {
        // Caught by ExceptionHandler, formatted as 422
        throw $e;
    } catch (\Exception $e) {
        $this->sendServerError($e->getMessage());
    }
}

Example 3: Service Layer Throwing Exceptions

// File: src/api/services/OrderService.php

public function cancelOrder(int $orderId): bool {
    // Find order
    $order = $this->orderRepo->findById($orderId);

    if (!$order) {
        // Service throws exception, controller catches it
        throw new NotFoundException("Order {$orderId} not found");
    }

    if ($order['status'] === 'cancelled') {
        // Business logic error
        throw new \Exception('Order is already cancelled');
    }

    // Cancel order...
    return $this->orderRepo->update($orderId, [
        'status' => 'cancelled',
        'cancelled_at' => date('Y-m-d H:i:s')
    ]);
}

Validation Helper

BaseController provides a validateRequest() helper:

protected function validateRequest(array $rules): array

// Usage:
$errors = $this->validateRequest([
    'email' => 'required|email',
    'name' => 'required|min:3',
    'age' => 'required|integer|min:18'
]);

if (!empty($errors)) {
    throw new ValidationException('Validation failed', $errors);
}

Best Practices

✅ DO:

  • Use framework exception classes (NotFoundException, ValidationException)
  • Throw exceptions from services/repositories, catch in controllers
  • Use appropriate HTTP status codes
  • Use sendSuccess() for successful operations
  • Use ValidationException for validation errors
  • Let ExceptionHandler format uncaught exceptions
  • Enable SHOW_ERRORS=true in development
  • Enable LOG_ERRORS=true to track issues
  • Return consistent JSON structure

❌ DON'T:

  • Don't return error arrays from services (throw exceptions instead)
  • Don't use wrong status codes (404 for validation errors, etc.)
  • Don't expose sensitive data in error messages
  • Don't show stack traces in production
  • Don't catch exceptions without re-throwing or handling properly
  • Don't use generic Exception for all errors (use specific types)
  • Don't log sensitive information (passwords, tokens)

Common Error Scenarios

Scenario 1: Resource Not Found

// In Repository or Service
$product = $this->productRepo->findById($id);
if (!$product) {
    throw new NotFoundException("Product not found");
}

Scenario 2: Validation Failed

// In Controller
$errors = $this->validateRequest($rules);
if (!empty($errors)) {
    throw new ValidationException('Validation failed', $errors);
}

Scenario 3: Database Error

// Automatically thrown by Database class
try {
    $result = $this->db->query([...]);
} catch (DatabaseException $e) {
    // Log and re-throw or handle
    throw $e;
}

Scenario 4: Business Logic Error

// In Service
if ($order['status'] === 'completed') {
    throw new \Exception('Cannot cancel completed order');
}

Environment Variables

Variable Values Purpose
SHOW_ERRORS true / false / 1 / 0 Show detailed error info (debug mode)
LOG_ERRORS true / false / 1 / 0 Write errors to log file
APP_ENV local / dev / production Environment mode
DEBUG_MODE true / false / 1 / 0 General debug flag

Recommended Settings

# Development (.env.local)
SHOW_ERRORS=true
LOG_ERRORS=true
APP_ENV=local
DEBUG_MODE=true

# Production (.env.production)
SHOW_ERRORS=false
LOG_ERRORS=true
APP_ENV=production
DEBUG_MODE=false

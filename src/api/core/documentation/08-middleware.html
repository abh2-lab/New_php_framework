Middleware System

Middleware allows you to execute code before and after your controller logic.

Think of middleware as "hooks" or "filters" that wrap around your controller execution, enabling cross-cutting concerns like authentication, logging, validation, and monitoring without cluttering controller code.

What is Middleware?

Middleware provides two execution hooks:

  • Before Middleware: Runs before the controller (can halt execution)
  • After Middleware: Runs after the controller (can modify response)

MiddlewareInterface

INTERFACE

All middleware must implement the MiddlewareInterface:

// File: src/api/core/middleware/MiddlewareInterface.php

namespace App\Core\Middleware;

interface MiddlewareInterface {
    /**
     * Execute before controller
     * @return mixed|null Return value halts execution and becomes response
     */
    public function before();

    /**
     * Execute after controller
     * @param mixed $response Controller response
     * @return void
     */
    public function after($response);
}

BaseMiddleware Abstract Class

ABSTRACT

The framework provides BaseMiddleware as a foundation for your middleware:

// File: src/api/core/middleware/BaseMiddleware.php

abstract class BaseMiddleware implements MiddlewareInterface {
    protected array $request;
    protected array $session;

    public function __construct() {
        // Parse JSON input
        $input = file_get_contents('php://input');
        $data = json_decode($input, true);

        $this->request = $data ?? $_POST;
        $this->session = $_SESSION ?? [];
    }

    /**
     * Helper to halt execution and return JSON response
     */
    protected function haltWithJson(int $statusCode, string $status, string $message) {
        http_response_code($statusCode);
        header('Content-Type: application/json');
        echo json_encode([
            'status' => $status,
            'message' => $message
        ]);
        exit;
    }

    // Must implement these in child classes
    abstract public function before();
    abstract public function after($response);
}

What BaseMiddleware Provides

  • $this->request: Parsed JSON or POST data
  • $this->session: Session data array
  • haltWithJson(): Stop execution and send JSON response

Creating Custom Middleware

Step 1: Create Middleware File

Create your middleware class in:

src/api/middleware/{YourMiddleware}.php

Step 2: Extend BaseMiddleware

// File: src/api/middleware/AuthMiddleware.php

namespace App\Middleware;

use App\Core\Middleware\BaseMiddleware;

class AuthMiddleware extends BaseMiddleware {

    public function before() {
        // Check for auth token
        $token = $_SERVER['HTTP_AUTHORIZATION'] ?? null;

        if (!$token) {
            // Halt execution with 401 response
            $this->haltWithJson(401, 'error', 'Unauthorized: Missing token');
        }

        // Validate token logic here...
        if (!$this->isValidToken($token)) {
            $this->haltWithJson(403, 'error', 'Forbidden: Invalid token');
        }

        // Token is valid, continue to controller
        return null; // null = continue execution
    }

    public function after($response) {
        // Nothing to do after controller
    }

    private function isValidToken(string $token): bool {
        // Your token validation logic
        return true;
    }
}

Step 3: Register Middleware

In src/api/routes.php, register your middleware with a name:

// Register middleware
$router->registerMiddleware('auth', \App\Middleware\AuthMiddleware::class);

Step 4: Attach to Routes

// Attach middleware to a single route
$router->add([
    'method' => 'GET',
    'url' => '/users/profile',
    'controller' => 'UserController@profile',
    'middleware' => 'auth', // Single middleware
    'visible' => true
]);

Before Middleware

BEFORE

Before middleware executes before the controller and can halt execution.

Use Cases

  • Authentication and authorization
  • Request validation
  • Rate limiting
  • Request logging
  • CORS preflight handling

Halting Execution

If before() returns a non-null value, execution stops and that value becomes the response:

public function before() {
    if (!$this->isAuthenticated()) {
        // Halt and send JSON response
        $this->haltWithJson(401, 'error', 'Not authenticated');
        // Code after this never runs
    }

    // Return null to continue to controller
    return null;
}

Real Example: BlockingMiddleware

// File: src/api/middleware/BlockingMiddleware.php

class BlockingMiddleware extends BaseMiddleware {

    public function before() {
        error_log("BlockingMiddleware: Blocking request");

        // Always block with 403
        $this->haltWithJson(403, 'error', 'Access forbidden by middleware');

        // Controller never executes
    }

    public function after($response) {
        // Never called because before() halts execution
    }
}

After Middleware

AFTER

After middleware executes after the controller completes.

Use Cases

  • Response logging
  • Adding response headers
  • Response transformation
  • Performance tracking
  • Analytics

Important: After middleware cannot halt execution. The response has already been generated by the controller.

Real Example: LogMiddleware

// File: src/api/middleware/LogMiddleware.php

class LogMiddleware extends BaseMiddleware {
    private float $startTime;

    public function before() {
        // Record start time
        $this->startTime = microtime(true);

        // Log incoming request
        error_log(sprintf(
            "Request: %s %s from %s",
            $_SERVER['REQUEST_METHOD'],
            $_SERVER['REQUEST_URI'],
            $_SERVER['REMOTE_ADDR']
        ));

        return null; // Continue to controller
    }

    public function after($response) {
        // Calculate request duration
        $duration = round((microtime(true) - $this->startTime) * 1000, 2);

        // Log response
        error_log(sprintf(
            "Response: %s ms",
            $duration
        ));
    }
}

Registering and Attaching Middleware

Registration in routes.php

First, register middleware with a short name:

// File: src/api/routes.php

// Register middleware
$router->registerMiddleware('log', \App\Middleware\LogMiddleware::class);
$router->registerMiddleware('auth', \App\Middleware\AuthMiddleware::class);
$router->registerMiddleware('admin', \App\Middleware\AdminMiddleware::class);

Single Middleware on Route

$router->add([
    'method' => 'GET',
    'url' => '/test/with-middleware',
    'controller' => 'TestController@testRoute',
    'middleware' => 'log', // Single middleware
    'visible' => true
]);

Multiple Middleware on Route

$router->add([
    'method' => 'POST',
    'url' => '/admin/users',
    'controller' => 'AdminController@users',
    'middleware' => ['log', 'auth', 'admin'], // Multiple middleware
    'visible' => true
]);

Middleware Execution Order

When multiple middleware are attached:

// Middleware: ['log', 'auth', 'admin'] BEFORE (in order): 1. LogMiddleware->before() 2. AuthMiddleware->before() 3. AdminMiddleware->before() 4. → Controller executes AFTER (reverse order): 5. AdminMiddleware->after() 6. AuthMiddleware->after() 7. LogMiddleware->after()

Why reverse order for after? This creates a proper "wrapping" effect, like Russian nesting dolls. The first middleware to execute before() is the last to execute after().

Shared Middleware in Route Groups

// Apply middleware to all routes in group
$router->group([
    'prefix' => '/admin',
    'middleware' => ['auth', 'admin'] // Applies to all routes in group
], function($router) {

    // Route: /admin/dashboard (has auth + admin middleware)
    $router->add([
        'method' => 'GET',
        'url' => '/dashboard',
        'controller' => 'AdminController@dashboard'
    ]);

    // Route: /admin/users (has auth + admin + log middleware)
    $router->add([
        'method' => 'GET',
        'url' => '/users',
        'controller' => 'AdminController@users',
        'middleware' => 'log' // Adds to group middleware
    ]);
});

Real Framework Examples

Example 1: TestMiddleware

Simple middleware demonstrating before/after hooks:

// File: src/api/middleware/TestMiddleware.php

class TestMiddleware extends BaseMiddleware {

    public function before() {
        error_log("TestMiddleware: BEFORE controller execution");

        // Continue to controller
        return null;
    }

    public function after($response) {
        error_log("TestMiddleware: AFTER controller execution");

        // Add custom header to response
        header('X-Middleware-Test: Passed');
    }
}

Example 2: Rate Limiting Middleware

class RateLimitMiddleware extends BaseMiddleware {

    public function before() {
        $ip = $_SERVER['REMOTE_ADDR'];
        $key = "rate_limit:{$ip}";

        // Check request count (example using file cache)
        $requests = $this->getRequestCount($key);

        if ($requests > 100) {
            // Rate limit exceeded
            $this->haltWithJson(429, 'error', 'Rate limit exceeded');
        }

        // Increment counter
        $this->incrementRequestCount($key);

        return null;
    }

    public function after($response) {
        // Add rate limit headers
        header('X-RateLimit-Limit: 100');
        header('X-RateLimit-Remaining: ' . (100 - $this->getRequestCount()));
    }
}

Example 3: JSON Response Wrapper

class JsonWrapperMiddleware extends BaseMiddleware {

    public function before() {
        // Nothing to do before
        return null;
    }

    public function after($response) {
        // Wrap all responses in a standard format
        if (is_array($response)) {
            header('Content-Type: application/json');
            echo json_encode([
                'timestamp' => date('Y-m-d H:i:s'),
                'data' => $response
            ]);
        }
    }
}

Accessing Request Data in Middleware

Via BaseMiddleware Properties

public function before() {
    // Access parsed JSON/POST data
    $userId = $this->request['user_id'] ?? null;

    // Access session data
    $isLoggedIn = $this->session['logged_in'] ?? false;

    // Access global $_SERVER
    $method = $_SERVER['REQUEST_METHOD'];
    $ip = $_SERVER['REMOTE_ADDR'];
    $token = $_SERVER['HTTP_AUTHORIZATION'] ?? null;
}

Via Global $_GET, $_POST, $_REQUEST

public function before() {
    // Query parameters
    $page = $_GET['page'] ?? 1;

    // Form data
    $username = $_POST['username'] ?? null;

    // Cookies
    $sessionId = $_COOKIE['session_id'] ?? null;
}

Best Practices

✅ DO:

  • Use before middleware for authentication and validation
  • Use after middleware for logging and response transformation
  • Keep middleware focused on a single responsibility
  • Return null from before() to continue execution
  • Use haltWithJson() to stop execution cleanly
  • Register middleware with descriptive names
  • Use route groups for shared middleware
  • Log middleware execution for debugging

❌ DON'T:

  • Don't put business logic in middleware (use services)
  • Don't try to halt execution in after() (won't work)
  • Don't perform heavy operations in middleware (impacts all requests)
  • Don't modify global state unless necessary
  • Don't forget to return null from before() when continuing
  • Don't use middleware for route-specific logic (put in controller)

⚠️ Performance:

  • Middleware runs on every request it's attached to
  • Keep middleware execution fast (< 10ms recommended)
  • Avoid database queries in middleware when possible
  • Use caching for expensive operations (auth token validation, etc.)

When to Use Before vs After

Common Middleware Patterns

Pattern 1: Timing/Performance Tracking

class PerformanceMiddleware extends BaseMiddleware {
    private float $startTime;
    private int $startMemory;

    public function before() {
        $this->startTime = microtime(true);
        $this->startMemory = memory_get_usage();
        return null;
    }

    public function after($response) {
        $duration = (microtime(true) - $this->startTime) * 1000;
        $memory = memory_get_usage() - $this->startMemory;

        header("X-Response-Time: {$duration}ms");
        header("X-Memory-Used: {$memory} bytes");
    }
}

Pattern 2: API Key Validation

class ApiKeyMiddleware extends BaseMiddleware {

    public function before() {
        $apiKey = $_SERVER['HTTP_X_API_KEY'] ?? null;

        if (!$apiKey) {
            $this->haltWithJson(401, 'error', 'API key required');
        }

        if (!$this->isValidApiKey($apiKey)) {
            $this->haltWithJson(403, 'error', 'Invalid API key');
        }

        return null;
    }

    public function after($response) {}
}

Pattern 3: CORS Headers

class CorsMiddleware extends BaseMiddleware {

    public function before() {
        // Handle preflight requests
        if ($_SERVER['REQUEST_METHOD'] === 'OPTIONS') {
            header('Access-Control-Allow-Origin: *');
            header('Access-Control-Allow-Methods: GET, POST, PUT, DELETE');
            header('Access-Control-Allow-Headers: Content-Type, Authorization');
            http_response_code(200);
            exit; // Stop here for OPTIONS
        }

        return null;
    }

    public function after($response) {
        header('Access-Control-Allow-Origin: *');
    }
}
